## 제품 불변조건

- 공모전 출품작은 사주팔자·자미두수·서양 점성술을 결합한 상담 MCP다.
- 공개 Tool은 `generate_fortune_context` 하나를 유지한다. 명확한 필요가 없으면 Tool을 나누지 않는다.
- 필수 입력은 연·월·일·시·분·성별·출생지다. 출생지는 반드시 질문한다.
- 계산 로직은 결정론적이어야 하며 기준 서비스 `https://sky.told.me/`의 결과와 일치해야 한다.
- 사용자의 질문이 범위를 결정한다. 예: “사주팔자만”이면 자미두수와 서양 점성술을 답변에 섞지 않는다.
- 질문이 Tool 인자에 누락되면 일반 전체 풀이용 기본 질문을 사용한다.
- 생년월일시, 성별, 출생지와 상담 결과를 저장하지 않는다.
- 카카오 로그인 정보가 MCP 인자로 자동 전달된다고 가정하지 않는다.

## 답변 생성 구조

1. 세 체계의 차트를 로컬에서 결정론적으로 계산한다.
2. 질문에 필요한 검증된 사실과 해석 규칙만 짧은 카드로 만든다.
3. 외부 LLM에는 질문과 카드만 보내 최종 한국어 상담문을 작성시킨다.
4. 사실 변경, 범위 이탈, 거절, 빈 응답, 잘림, 지연은 검증하고 결정론적 fallback을 사용한다.
5. MCP가 완성된 답변을 반환하면 카카오 호스트 LLM에는 요약하지 말고 그대로 전달하도록 지시한다.

외부 LLM에 원문 차트 전체나 직접적인 출생 입력 요약을 보내지 않는다. 쉬운 표현을 쓰되 `일간`, `십신`, `대운`, `세운` 같은 근거를 함께 설명해 전문성을 유지한다.

## 보안과 운영 한도

- 비밀값을 소스, Git 기록, Docker 이미지, 프롬프트, 응답, 로그에 넣지 않는다.
- `CEREBRAS_API_KEY`는 로컬의 무시된 `.env`와 PlayMCP in KC의 시크릿 필드에만 둔다.
- 현재 생성기는 Cerebras `gpt-oss-120b`다. 무료 플랜은 5 RPM이므로 심사·데모 용량으로만 취급한다.
- 공개 트래픽 또는 본선 투표 전에 운영자 소유의 유료 추론 용량으로 전환해야 한다.
- 응답은 Markdown `TextContent`만 사용하고 24k 제한과 p99 3초 제약을 의식한다.

## 검증과 재배포

- 기본 검증: `npm test`, `npm run build`, `git diff --check`.
- 대표 fixture는 `1988-04-19 08:30`, 남성, 서울이다.
- GitHub push는 기존 KakaoCloud 서버를 갱신하지 않는다. 재배포는 새 Git 소스 서버를 만드는 방식이다.
- 재배포 순서:
  1. 최신 `main`으로 교체 서버를 만든다.
  2. 새 endpoint에서 initialize, `tools/list`, `tools/call`을 검증한다.
  3. 기존 PlayMCP 임시 등록을 수정해 새 endpoint를 정보 불러오기 후 저장한다. 임시 등록을 하나 더 만들지 않는다.
  4. PlayMCP AI Chat에서 “내 사주를 봐줘”와 출생 정보 입력의 2단계 흐름을 검증한다.
  5. 모든 검증 후에만 이전 서버를 삭제한다.
  6. README와 배포·제출 SSOT를 현재 상태로 갱신한다.

배포 환경변수와 상세 명령은 `docs/testing-and-deployment.md`를 따른다.
